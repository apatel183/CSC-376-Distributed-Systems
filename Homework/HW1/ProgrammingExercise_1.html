Programming Exercise

Program Input

Due: Mon., Apr. 3, at 10:10am

  • Functionality

    You are to create a program that properly handles two types of input-- command line arguments and standard input. The purpose of this exercise is to ensure that you can effectively handle both types of input and that you clearly understand the differences. In particular, the program will perform as follows:

    1. There are three possible command line arguments for this program: option1, option2, and option3. Each option appears following a single dash ('-'). A word of text follows certain options. Your program must read the actual arguments present and store any additional data associated with them.
    2. The options are abbreviated on the command line as follows:
      • option1: -o
      • option2: -t
      • option3: -h
    3. A word always follows option1 and option2. There is no data following option3.
    4. Any, all, or none of these options may be present on the command line; moreover, they may be present in any order. Your program must properly handle all possible combinations. For example, option3 (-h) may be present, followed by option1 (-o), while option2 is not present on the command line.
    5. In addition to command line arguments, your program must also read data from standard input.
    6. Your program will read data from standard input one line at a time. Your program must continue reading until there is no more data from standard input.
    7. As your program reads a line from standard input, it will output the line immediately to standard output.
    8. Once all data has been read from standard input and output, your program will display a list of the command line options found, along with their data, in numeric order. In other words, option1 must be displayed first, if present, then option2, and finally option3. DO NOT DISPLAY AN OPTION IF IT IS NOT PRESENT ON THE COMMAND LINE.
    9. If you write your program using Python, you can use the getopt.getopt function, which will do much of the heavy lifting for you.

  • Testing and Performance

    The following files are included to test whether your code works correctly. The scripts assume that the name of your Python code is input_program.py. If you choose to write in Java, the scripts assume that the name of your Java code is InputProgram.java and that you do not create your code within a package.

    • lines.txt -- a simple text file that is redirected to standard input
    • test_py_input.bat or test_py_input.sh -- a simple shell script that starts the input_program.py Python program and redirects lines.txt as the standard input; the .bat version is designed to run on Microsoft Windows, while the .sh version is designed for Mac OS X and Linux
    • test_java_input.bat or test_java_input.sh -- a simple shell script that starts the InputProgram Java program and redirects lines.txt as the standard input; the .bat version is designed to run on Microsoft Windows, while the .sh version is designed for Mac OS X and Linux
    • ref.txt -- a reference version of the expected output using the above files; your output should match it if your code works correctly
    • PerformanceRubric_ProgrammingExercise1.txt -- a rubric for analyzing the correctness of your output
